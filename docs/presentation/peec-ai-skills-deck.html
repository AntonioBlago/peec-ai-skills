/peec-setup — 9 phases, full-funnel coverage.

Takes a Peec project from empty (or broken) to operator-ready. Owns setup_state.json — every other skill refuses to run without it.

When to invoke

  • "Set up Peec for <client>"
  • "My Peec competitors are wrong"
  • "Design prompts for my customer journey"
  • "Restructure topics / tags"
  • Audit of an existing tracking setup

Scope modes

scope=full                   # greenfield, all 9 phases
scope=audit                  # live-diff, re-run drifted phases
scope=partial:gsc_mapping    # jump to one phase
scope=import                 # brownfield reconstruction

The 9 phases

  1. Phase 0 — state check & mode selection
  2. Competitor discovery from real AI chats (not Google)
  3. Customer-journey prompts: Awareness → Consideration → Decision → Retention
  4. Topic taxonomy (one topic per prompt, funnel-aligned)
  5. Tag taxonomy (cross-cut: branded/non-branded/informational/transactional/…)
  6. GSC keyword mapping via Visibly
  7. Forum pain-point mining (Reddit, Gutefrage, t3n, OMR)
  8. Hero prompt definition
  9. P0/P1/P2 backlog + state write
Never silently defaults to EN/en. Country + language travel through every downstream skill via setup_state.json.

/peec-cluster — zones, not keyword groups.

Reduces a flat prompt set to 4–8 strategic zones, each with one structural weakness in competition and one measurable metric. Output is a strategic map — not a content calendar.

A zone is valid only when all three axes line up

      Intent
       ↑
Visibility gap
       ↑
Demand signal

If a candidate zone fails any axis → merged or dropped.

When to invoke

  • ≥20 Peec prompts — team has lost the overview
  • Quarterly strategy refresh
  • Before an investment conversation ("what is our AI content strategy?")

Do not use with <10 prompts or when funnel stages aren't set.

Output — one zone map + Peec tags

  • Markdown zone map — one zone per section, each with: structural weakness · measurable metric · exact next action
  • Peec tag per zone (zone:<slug>) — all prompts retagged so /peec-report can measure zone lift later

/peec-learn — cross-project memory layer.

Project-local learnings live in growth_loop/learnings.json. This skill promotes them across projects: lessons from project A become priors for project B.

Two modes

  • Write — extract 1–3 patterns from a just-produced artifact (decision, brief, zone map, outreach log, learnings.json) and persist to SkillMind.
  • Read — on orchestrator start, recall patterns matching gap type / project / skill → hand back as priors.

Pattern quality bar

  • Causal — X caused Y, not correlation
  • Falsifiable — states the condition for being wrong
  • Evidence-backed — links the artifact that produced it

No speculation. Artifact must have a measured outcome.

Backend

Pinecone-backed semantic memory via mcp__skillmind__*.
Fallback when MCP unavailable: append to <project>/growth_loop/patterns.md.

Example pattern (write mode)

{

    "project_id": "or_bf5b...",

Install in 30 seconds.

One bash script symlinks all 9 skills into ~/.claude/skills/. git pull in the repo updates every installed skill in place.

Default install (symlink)

git clone https://github.com/AntonioBlago/peec-ai-skills.git ~/peec-ai-skills
cd ~/peec-ai-skills
./claude-peec-ai.sh                              # symlink all 9 skills

Useful flags

./claude-peec-ai.sh --copy                       # copy instead of symlink (no git pull updates)
./claude-peec-ai.sh --target ./.claude            # per-project install
./claude-peec-ai.sh --force                       # overwrite existing skill dirs
./claude-peec-ai.sh --dry-run                     # preview, no fs changes
./claude-peec-ai.sh --uninstall                   # remove all skill entries

On Windows: symlinks need Developer Mode or admin rights. Use --copy as a safe fallback.

Phase 0 mode selection — what actually runs.

Cheap (one file read + at most one parallel Peec read). Decides whether the rest of the setup is needed at all.

State fileLive Peec contentModeWhat runs
MissingEmpty (≤2 brands AND ≤4 prompts AND ≤0 topics)fullAll 9 phases. Greenfield.
MissingPopulated (≥3 brands OR ≥5 prompts OR ≥1 topic)importReconstruct state, ask 3 questions, exit. No discovery.
Present, < 30 days—skipShow summary, ASK before re-running.
Present, 30–90 days—auditLive-diff snapshot. Re-run only phases with >20% drift.
Present, > 90 days—full (warn)Recommend full re-run; data is stale.

User can always force a mode

/peec-setup scope=full              # ignore state, re-run from scratch
/peec-setup scope=audit             # force audit even on fresh state
/peec-setup scope=partial:gsc_mapping   # jump to one specific phase
